# Guia de demo para cliente

## Objetivo

Mostrar que el sistema permite manejar reservas reales de una peluqueria/barberia con dos vistas separadas: cliente final y peluquero/admin.

## Credenciales demo

Admin:

- correo: `admin@peluqueria.com`
- clave: `Admin123*`

Cliente:

- correo: `cliente@demo.com`
- clave: `Cliente123*`

## Flujo cliente

1. Iniciar sesion como cliente.
2. Ver estado del peluquero.
3. Elegir servicio.
4. Elegir fecha.
5. Seleccionar hora disponible.
6. Ver cita creada en "Mis citas".
7. Ver ubicacion y QR de pago.

## Flujo admin

1. Iniciar sesion como admin.
2. Ver resumen de citas e ingresos.
3. Ver cita creada por el cliente.
4. Cambiar estado de cita a confirmada.
5. Marcar pago como abonado o pagado.
6. Revisar historial de pagos.
7. Crear o revisar servicios.
8. Editar datos del negocio si aplica.

## Mensaje para presentar

Esta es una prueba piloto funcional. Permite validar si los clientes realmente usan reservas web antes de invertir en pagos automaticos, WhatsApp o funciones avanzadas.

## Que no se debe prometer todavia

- Pago automatico con Wompi.
- Notificaciones automaticas por WhatsApp.
- App movil en Play Store o App Store.
- Facturacion electronica.
- Reportes avanzados.
- Multiples peluqueros.

Estas funciones se pueden plantear como fases adicionales despues de validar la prueba piloto.
